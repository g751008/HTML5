網際網路(Internet) = 把家庭內的少人數網路和企業使用的大規模網路等不同種類網路互相連結起來的世界級網路，以一組標準的網路TCP/IP協定族相連。

WWW(World Wide Web) = 其中一種存取網際網路上的資訊的機制，只要存取畫面上的超連結就能前往網路上的各種地方(此機制稱為超文件系統 Hypertext System)。

HTML(Hyper Text Markup Language) = 用來寫網頁的標籤語言，以<>標籤框住來表現區塊。
1990由隸屬CERN(歐洲核子研究機構)的Tim Berners-Lee開發，同一時期HTTP、URI、WWW、世上第一個Web瀏覽器也被開發出來。當時被用於CERN內部共享研究成果，1993年CERN正式發表WWW並推動網際網路連線服務後，被確立為網際網路上的文件標準規範。
HTML最初為研究者專用，1993年隸屬於伊利諾大學的NCSA(國家超級電腦應用中心)的Mark Anderson開發的Web瀏覽器Mosaic能同時顯示文字與圖片，遠比以往Web瀏覽器方便，在此之後登場的Netscape Navigator和Internet Explorer，讓網際網路在一般使用者之前普及。
HTML最早由IETF(Internet Engineering Task Force)制定標準，3.2後由Tim Berners- Lee組建的W3C負責。1993:HTML1.0 1995:HTML2.0 1997:HTML3.2 1997:HTML4.0 1999:HTML4.1。 之後W3C制定XHTML來取代HTML。但制定2.0版前宣告終止，而將心力放在制定HTML5上。

Web伺服器 = 負責回應Web瀏覽器要求(Request)並提供服務回應(Response)的電腦，藉由HTTP(Hyper Text Transfer Protocol)和Web瀏覽器進行通訊，除傳送HTML網頁也可傳送相關圖片或音訊內容。

動態網頁 = 可依用戶端要求執行程式並傳送結果的網頁(另一方面，只能傳回固定內容則稱靜態網頁)。

伺服端網頁技術是在Web伺服器上執行的應用程式，而不是在客戶端瀏覽器執行，有下列幾種：
共通閘道介面CGI(Common Gateway Interface) = 依照來自Web瀏覽器的要求啟動Web伺服器上的程式的機制，用於CGI的程式是以Perl或C語言之類的程式語言所寫成，需先編譯成可執行檔，以便在伺服端執行。
ASP(Active Server Pages) = 名稱直譯是一種讓網頁在伺服器上動起來的技術，能夠讓Script語言直接內嵌網頁，在伺服端產生動態的網頁內容並以直譯方式執行。
ASP.NET伺服端網頁技術屬於微軟.NET Framework技術的成員，以CLR(Common Language Runtime)架構的.NET程式設計平台，可以讓我們使用CLR語言在伺服端建立Web應用程式。
.NET Framework是微軟的程式開發平台，由CLR和.NET Framework類別函數庫(.NET Framework Class Library，.NET FCL，提供龐大類別以供建立各種不同的應用程式，有支援.NET Framework的語言都可以使用)組成，使用.NET Framework支援的程式語言編寫的程式碼可使用.NET編譯程式進行編譯成MSIL(Microsoft Intermediat Language，一種中間程式語言而非CPU可執行的機器語言)，執行程式時CLR再使用JIT(Just In Time)編譯程式將MSIL轉換成機器語言來執行，因此只須作業系統安裝CLR即可跨Windows平台執行。

PHP(PHP:Hypertext Preprocessor):一種開放原始碼(Open Source)的伺服端Script語言，可內嵌於HTML網頁，主要使用在Linux/Unix作業系統的伺服端網頁技術，目前也支援Windows作業系統。

JSP(Java Server Pages):Java家族中和ASP一較長短的網頁技術，結合HTML和Java Servlet的伺服端網頁技術，以Java來說，Java Applet是下載到客戶端執行的程式檔，Java Servlet是在伺服端執行。

伺服器端腳本(Server Side Script) = 先在Web伺服器端的HTML檔案中寫上腳本(Script;簡單的程式)，再將執行結果作為HTML一部分傳回給Web瀏覽器，此為伺服器端腳本。

用戶端腳本 = 相對於伺服器端腳本，在Web瀏覽器上執行的腳本。主流為JavaScript。
JavaScript是Netscape開發的一種一列一列可執行的腳本語言，可內嵌HTML網頁成為網頁一部分，目前是甲骨文公司的註冊商標，使用類似Java語言的語法，是源自於Lisp語言的一種原型基礎的物件導向程式語言，而Jscript為微軟推出相容JavaScript的腳本語言。

DOM物件模型(Object Model)將HTML元素轉換成節點樹，讓我們走訪節點存取HTML元素，由DOM Core(提供網頁或XML瀏覽(Navigator)走訪節點或參考(Reference)存取節點的集合物件)和DOM HTML(網頁專屬的DOM API介面，讓JavaScript存取元素建立動態網頁內容)組成。

jQuery是一個JavaScript函數庫，提供簡潔方式來撰寫JavaScript和擴充JavaScript功能。是2006年John Resig在BarCamp NYC發表的網頁技術，目前是MIT和GPL授權的免費軟體。所有jQuery做的事傳統JavaScript API也做得到，只是jQuery做起來簡單的多。

HTML5 = W3C(World Wide Web Consortium)制定的HTML新標準，包含CSS3(Cascading Style Sheets 3)和JavaScript API(Application Programming Interface)，比XHTML語法鬆散，因為它不是XML語言不用導循XML語法。例如:html、head、body標籤可有可無，元素和屬性不分大小寫，結尾標籤可省略如 [p]段落 或 [br]，屬性值引號可有可無如[img width=20]，沒有屬性值的屬性只需使用屬性名稱如[option selected]，文字可以不用存在於HTML開始結束標籤之間，一些舊版HTML和XHTML的屬性不再需要如script的type屬性或html的xmlns屬性等。
HTML5能取代HTML4.01並對應越來越強大的Web應用程式和Web內容，將繪圖和不會伴隨網頁遷移的通訊等之前靠Adobe Flash和Microsoft Silverlight之類外掛程式實現的功能放入標準規範。為了實現上述規範，JavaScript準備了專用的物件、屬性和方法。這些擴充後的物件稱為API(Application Programming Interface)，支援到哪個地步會因瀏覽器而異。

IE (Internet Explorer) = 微軟開發的Web瀏覽器，隨附於Windows。

Google Chrome = Google開發的Web瀏覽器，可與Google服務接軌。

Mozilla Firefox = Mozilla foundation開發的Web瀏覽器，有強大的擴充性。

Opera = Opera Software開發的Web瀏覽器，有豐富的功能。

Safari = 蘋果公司開發的Web瀏覽器，能與Mac、iPhone等Apple產品接軌。

AJAX = Asynchronous JavaScript + XML(非同步JavaScript + XML)的簡稱，是JavaScript融合CSS、HTML和XML。原只是Google Map使用的技巧，2005年Jesse James Garrett發表的文章正式出現AJAX這個名詞。 關鍵是藉由XMLHttpRequest這個JavaScript物件與伺服器進行非同步通訊。

CSS(Cascading Style Sheets)簡稱樣式表，用來在網頁中指定設計或佈局之類的文件外觀相關部分的機制，1996年公布CSS Level1，1998年推出CSS Level2，2011年推出CSS Level3。
HTML負責描述文件內容與意義，CSS負責外觀裝飾。

HTML標籤的共同屬性:id(元素唯一識別名稱不能重複)、accesskey(指定元素的快捷鍵來取得焦點)、class(指定元素套用的樣式類別)、dir(指定元素內容文字方向從左到右或從右到左ltr、rtl、auto)、lang(指定元素使用的語言)、style(指定元素套用的局部套用CSS)、tabindex(指定按下tab鍵移動元素取得焦點的順序)、title(指定元素的額外資訊)
